# RAG 知识检索模块设计

## 整体流程

```
用户 query
    │
    ├──→ BM25 关键词检索 ──→ 得分排序 ──┐
    │                                   ├──→ RRF 融合 → Top-3 → Prompt 注入 → LLM 回答
    └──→ ANN 向量检索 ────→ 得分排序 ──┘
```

## 两路检索

### BM25 关键词检索

标准 BM25 算法，参数 k1 = 1.5, b = 0.75。

- **分词**：按空白符切分 query，对每个 term 独立计算
- **文档频率**：统计每个 term 在文档集合中出现的文档数
- **得分**：对各 term 的 BM25 得分求和，逆文档频率采用标准平滑公式

当前在 `localDocs` 列表中检索 4 篇预先定义的领域知识文档。

### ANN 向量检索

基于 IVF（Inverted File Index）的近似最近邻搜索。

- **索引构建**：k-means 聚类将向量空间划分为 4 个簇，每个 DocumentChunk 分配到最近的簇
- **检索策略**：定位 query 最近的 2 个簇，仅在这两个簇内遍历计算余弦相似度
- **降级策略**：当 ANN 结果不足 topK 时，自动回退到全量暴力搜索
- **Embedding 模型**：默认使用本地 Ollama 的 nomic-embed-text 模型，128 维输出向量

## RRF 融合排序

使用互惠排名融合（Reciprocal Rank Fusion）合并两路结果：

```
score(d) = Σ 1 / (k + rank(d))
```

k = 60，分别计算每个文档在两路结果中的排名得分并求和，取总分最高的 Top-3。

## 领域知识分类

| 分类 | 覆盖内容 | 文档数量 |
|------|---------|---------|
| 活动规则 | 满减促销、优惠券叠加规则 | 1 |
| 售后政策 | 七天无理由、质量问题处理 | 1 |
| 物流说明 | 出库时效、配送范围 | 1 |
| 商品导购 | 品类选购建议、场景推荐 | 1 |

## 向量存储说明

`InMemoryVectorStore` 运行时存储在内存中，重启后需重新构建索引。每次写入或删除操作会使索引标记为"脏"，下次检索时自动重建。

## 降级策略

| 故障场景 | 降级行为 |
|---------|---------|
| Ollama 服务不可用 | 回退到基于文本 hashCode 的随机向量 |
| ANN 结果不足 | 全量暴力搜索兜底 |
| 文档集合为空 | 跳过 RAG 注入，仅使用模型自身知识 |
